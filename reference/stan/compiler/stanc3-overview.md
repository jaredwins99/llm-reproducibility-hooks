## A New Stan-to-C++ Compiler

stanc3 is a compiler for Stan written in OCaml. Since version 2.26, it has been the default compiler for Stan. The latest release with binaries for many platforms and a JavaScript version is available on the GitHub releases page.

## Documentation

- **User documentation**: Located in the Stan Users' Guide at https://mc-stan.org/docs/stan-users-guide/using-the-stan-compiler.html
- **Developer documentation**: Available at https://mc-stan.org/stanc3/stanc

## High-level Architecture

stanc3 contains 4 main source packages coordinated by the `driver` module:

1. **frontend** -- Lexing, parsing, and type checking
2. **middle** -- Contains the MIR (Middle Intermediate Representation) and shared types
3. **analysis_and_optimization** -- Dataflow analysis and optimization passes
4. **stan_math_backend** -- C++ code generation

The architecture maintains as many details as possible about Stan's C++ implementation in the Stan Math backend library. The compiler's entry point is `src/stanc/stanc.ml`, which parses command-line arguments and calls `src/driver/Entry.ml` to sequence the components.

## Distinct stanc Phases

1. **Lexing**: Convert Stan source into tokens via `src/frontend/lexer.mll`
2. **Parsing**: Generate AST from tokens using `src/frontend/parser.mly` (use `stanc --debug-ast` to view)
3. **Type Checking**: Add type information via `src/frontend/Typechecker.ml` (use `stanc --debug-decorated-ast`)
4. **Lowering**: Convert to MIR via `src/frontend/Ast_to_Mir.ml` (use `stanc --debug-mir` or `--debug-mir-pretty`)
5. **Backend Transform**: Apply backend-specific MIR transformations via `src/stan_math_backend/Transform_Mir.ml` (use `stanc --debug-transformed-mir`)
6. **Analysis & Optimization**: Improve MIR representation
7. **Code Generation**: Generate C++ or other outputs via `src/stan_math_backend/Lower_program.ml`

## Central Data Structures

### AST (`src/frontend/Ast.ml`)
Direct 1-1 mapping with Stan syntax, preserving elements like parentheses. Uses "two-level types" pattern for metadata attachment. The pretty-printer maintains user syntax while adjusting whitespace.

### MIR (`src/middle/Program.ml`)
Generated by `src/frontend/Ast_to_Mir.ml`, strips Stan-specific semantics. Also employs two-level types for expression types, autodiff levels, and location metadata. Used as input for dataflow analysis, optimization, and code generation.

### C++ Representation (`src/stan_math_backend/Cpp.ml`)
Minimal C++ representation tailored specifically to the model class generation, simpler than both AST and MIR.

## Design Goals

- **Multiple phases** with human-readable intermediate representations enabling straightforward debugging and optimization development
- **Optimizing** approach leveraging Stan language-level information, reducing user burden for writing performant code
- **Holistic** architecture bringing maximum code into MIR for whole-program optimization
- **Research platform** enabling novel optimizations based on probability theory
- **Modular** design facilitating external library integration and tool development around Stan
- **Simplicity first** philosophy: choose correctness and simplicity over perceived performance gains unless benchmarks demonstrate >5% improvements

## Main Code Modules

Seven major code sections:
- **Driver** -- Entry point and phase sequencing
- **Frontend** -- Lexer, parser, type checker
- **Middle** -- MIR and shared types
- **Common** -- Shared utilities
- **Analysis_and_optimization** -- Dataflow and optimization
- **Stan_math_signatures** -- Stores C++ library signatures exposed to Stan
- **Stan_math_backend** -- C++ code generation
