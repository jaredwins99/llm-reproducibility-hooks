# Posterior Predictive Sampling - Complete Content

## Overview

The posterior predictive distribution represents evaluating or sampling from p(ỹ|y), where y is observed data and ỹ is yet-to-be-observed data. With predictors, this becomes p(ỹ|x̃,x,y).

## Posterior Predictive Distribution

Given a full Bayesian model p(y,θ), the posterior predictive density for new data ỹ given observed data y is:

$$p(\tilde{y} \mid y) = \int p(\tilde{y} \mid \theta) \cdot p(\theta \mid y) \, d\theta$$

This integral marginalizes out parameters θ, leaving the predictive density of future observations given past observations.

## Computing the Posterior Predictive Distribution

Using M Monte Carlo draws θ^(m) ~ p(θ|y) from the posterior:

$$p(\tilde{y} \mid y) \approx \frac{1}{M} \sum_{m = 1}^M p(\tilde{y} \mid \theta^{(m)})$$

The log posterior predictive density can be computed using:

$$\log p(\tilde{y} \mid y) \approx - \log M + \text{log-sum-exp}_{m = 1}^M \log p(\tilde{y} \mid \theta^{(m)})$$

where:

$$\text{log-sum-exp}_{m = 1}^M v_m = \log \sum_{m = 1}^M \exp v_m$$

## Sampling from the Posterior Predictive Distribution

Given posterior draws θ^(m) ~ p(θ|y), draws from the posterior predictive ỹ^(m) ~ p(ỹ|y) are generated by:

$$\tilde{y}^{(m)} \sim p(y \mid \theta^{(m)})$$

This approach captures two uncertainty types:

**Aleatoric uncertainty** arises from the data model p(ỹ|θ) with unknown θ.

**Epistemic uncertainty** comes from θ being estimated from finite data.

The factored form shows both:

$$p(\tilde{y} \mid y) = \int \underbrace{p(\tilde{y} \mid \theta)}_{\text{aleatoric}} \cdot \underbrace{p(\theta \mid y)}_{\text{epistemic}} \, d\theta$$

## Simple Poisson Model Example

Consider a Poisson model with rate parameter λ > 0:

$$\lambda \sim \text{gamma}(1, 1)$$
$$y_n \sim \text{poisson}(\lambda)$$

### Stan Code

```stan
data {
  int<lower=0> N;
  array[N] int<lower=0> y;
}
parameters {
  real<lower=0> lambda;
}
model {
  lambda ~ gamma(1, 1);
  y ~ poisson(lambda);
}
generated quantities {
  int<lower=0> y_tilde = poisson_rng(lambda);
}
```

The `poisson_rng` function generates random draws accounting for aleatoric uncertainty. Stan's posterior sampler handles epistemic uncertainty by generating ỹ^(m) ~ p(y|λ^(m)) for each posterior draw λ^(m).

## Computing Expectations

In general, E[f(ỹ,θ)|y] is evaluated as:

$$\mathbb{E}[f(\tilde{y}, \theta) \mid y] \approx \frac{1}{M} \sum_{m=1}^M f(\tilde{y}^{(m)}, \theta^{(m)})$$

This is the posterior mean of f(ỹ,θ) computed by Stan.

Example:
```stan
generated quantities {
  real f_val = f(y_tilde, theta);
}
```

## Analytic Posterior and Posterior Predictive

For the Poisson example, since the gamma distribution is the conjugate prior for the Poisson distribution, the posterior p(λ|y) follows a gamma distribution.

The posterior predictive p(ỹ|y) follows a negative binomial distribution (the compound gamma-Poisson distribution). Rather than analytically marginalizing λ, the procedure samples λ^(m) ~ p(λ|y) and generates ỹ^(m) ~ p(y|λ^(m)).

## Posterior Prediction for Regressions

### Posterior Predictive Distributions for Regressions

For a regression with predictor x_n for training outcome y_n and x̃_n for test outcome ỹ_n:

$$p(\tilde{y} \mid \tilde{x}, y, x) = \int p(\tilde{y} \mid \tilde{x}, \theta) \cdot p(\theta \mid y, x) \, d\theta \approx \frac{1}{M} \sum_{m=1}^M \, p(\tilde{y} \mid \tilde{x}, \theta^{(m)})$$

where θ^(m) ~ p(θ|x,y).

### Stan Program for Poisson Regression

```stan
data {
  int<lower=0> N;
  vector[N] x;
  array[N] int<lower=0> y;
  int<lower=0> N_tilde;
  vector[N_tilde] x_tilde;
}
parameters {
  real alpha;
  real beta;
}
model {
  y ~ poisson_log(alpha + beta * x);
  { alpha, beta } ~ normal(0, 1);
}
generated quantities {
  array[N_tilde] int<lower=0> y_tilde
    = poisson_log_rng(alpha + beta * x_tilde);
}
```

The model fits α and β using training data x, while the generated quantities block generates predictions ỹ for test data x̃. This produces a predictive sample ỹ^(1),...ỹ^(M) where each ỹ^(m) ~ p(ỹ|x̃,x,y).

## Posterior Predictive Mean

The mean of the posterior predictive distribution is:

$$\mathbb{E}[\tilde{y} \mid \tilde{x}, x, y] = \int \tilde{y} \cdot p(\tilde{y} \mid \tilde{x}, \theta) \cdot p(\theta \mid x, y) \, d\theta \approx \frac{1}{M} \sum_{m = 1}^M \tilde{y}^{(m)}$$

The posterior mean of `y_tilde[n]` after running Stan is the expected value of ỹ_n conditioned on training data. This is the Bayesian estimate with minimum expected squared error.

Posterior draws also estimate quantiles, posterior intervals, and covariances. All calculations require only a single Stan run producing M draws.

## Estimating Event Probabilities

Event probabilities involving parameters or predictions are coded in the generated quantities block.

### Simple Example: Parameter Probability

To evaluate Pr[λ > 5|y] in the Poisson example:

```stan
generated quantities {
  int<lower=0, upper=1> lambda_gt_5 = lambda > 5;
}
```

The expression `lambda > 5` equals 1 if true, 0 otherwise. The posterior mean of this parameter is the event probability:

$$\Pr[\lambda > 5 \mid y] = \int \mathbb{I}(\lambda > 5) \cdot p(\lambda \mid y) \, d\lambda \approx \frac{1}{M} \sum_{m = 1}^M \mathbb{I}[\lambda^{(m)} > 5]$$

where each λ^(m) ~ p(λ|y). Stan recovers this as the posterior mean of `lambda_gt_5`.

### General Event Probabilities

Event probabilities are expectations of indicator functions:

$$\Pr[\lambda > 5 \mid y] = \mathbb{E}[\mathbb{I}[\lambda > 5] \mid y] = \int \mathbb{I}(\lambda > 5) \cdot p(\lambda \mid y) \, d\lambda \approx \frac{1}{M} \sum_{m = 1}^M \mathbb{I}(\lambda^{(m)} > 5)$$

### Comparative Event Probabilities

For posterior predictive quantities, event probabilities work identically. For example, if ỹ_n is the prediction for the n-th unobserved outcome:

$$\Pr[\tilde{y}_3 > \tilde{y}_7 \mid \tilde{x}, x, y] = \mathbb{E}\!\left[\mathbb{I}[\tilde{y}_3 > \tilde{y}_7] \mid \tilde{x}, x, y\right] = \int \mathbb{I}(\tilde{y}_3 > \tilde{y}_7) \cdot p(\tilde{y} \mid \tilde{x}, x, y) \, d\tilde{y} \approx \frac{1}{M} \sum_{m = 1}^M \mathbb{I}(\tilde{y}^{(m)}_3 > \tilde{y}^{(m)}_7)$$

where ỹ^(m) ~ p(ỹ|x̃,x,y).

## Stand-Alone Generated Quantities and Ongoing Prediction

Stan's sampling algorithms produce posterior draws θ^(1),...,θ^(M) from p(θ|y,x). Posterior predictive draws ỹ^(m) ~ p(ỹ|x̃,x,y) are generated by drawing:

$$\tilde{y}^{(m)} \sim p(y \mid \tilde{x}, \theta^{(m)})$$

This depends only on new predictors x̃ and posterior draws θ^(m). The original data and model density are not required.

By saving posterior draws, predictions for new data x̃ can be generated whenever needed. A second Stan program inputs the original program's parameters and new predictors.

### Original Fitting Program

```stan
data {
  int<lower=0> N;
  vector[N] x;
  vector[N] y;
}
parameters {
  real alpha;
  real beta;
  real<lower=0> sigma;
}
model {
  y ~ normal(alpha + beta * x, sigma);
  alpha ~ normal(0, 5);
  beta ~ normal(0, 1);
  sigma ~ lognormal(0, 0.5);
}
```

### Stand-Alone Prediction Program

```stan
data {
  int<lower=0> N_tilde;
  vector[N_tilde] x_tilde;
}
parameters {
  real alpha;
  real beta;
  real<lower=0> sigma;
}
generated quantities {
  vector[N_tilde] y_tilde
    = normal_rng(alpha + beta * x_tilde, sigma);
}
```

The follow-on program declares only necessary data (predictors for new outcomes), original parameters, and predictions in the generated quantities block. No model or redundant parameters are needed.

Inputs required are the original posterior parameter draws and new predictors. Outputs are draws for ỹ or derived quantities such as event probabilities. Any posterior predictive quantities can be generated this way, including event probabilities coded as indicator variables.
